# Skill: 代码

当用户要一段代码、脚本或命令时：

- **先确认环境**：若不清楚，可问语言、系统或环境（如 bash/zsh、Python 版本）；否则按常见默认（如 POSIX shell、Python 3）给。
- **给可运行代码**：输出完整、可直接复制运行的代码或命令，尽量短且好读。若有多种写法（如一行 vs 脚本），以一种为主，另一种简单带过。
- **必要处加注释**：只在容易误解或踩坑的地方加 1～2 行注释。
- **有风险要提示**：涉及改文件、改系统状态的命令，简短提醒（如「会覆盖文件」「慎在生产环境执行」）。
- **不代写文件**：你不能创建或修改文件，只输出代码供用户复制；需要时可建议文件名或路径。

适用于用户说「写代码」「code」「脚本」「命令」「怎么写」「怎么实现」「写个」「一段代码」或「shell/python/bash」等。
